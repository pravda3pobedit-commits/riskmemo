# Tally Form Spec

Use this to create the first Tally intake form. Keep it short; the goal is to receive enough context without asking for sensitive personal data.

## Form Title

Разбор подозрительного сообщения

## Intro Text

Пришлите подозрительное сообщение, вакансию или фрагмент переписки. Я отмечу тревожные признаки, признаки шаблона/подмены, вопросы для проверки и безопасный вариант ответа.

Перед отправкой замажьте имена, телефоны, адреса, реквизиты, документы и другие личные данные.

## Fields

1. **Email для ответа**
   - Type: Email
   - Required: yes

2. **Что это за ситуация?**
   - Type: Multiple choice
   - Required: yes
   - Options:
     - Вакансия / работодатель
     - Фриланс-клиент / заказчик
     - Инвестиции / крипто / быстрый заработок
     - Знакомство / отношения
     - Партнерство / бизнес-предложение
     - Другое

3. **Текст сообщения или переписки**
   - Type: Long text
   - Required: yes
   - Help text: Вставьте 1-30 сообщений. Личные данные лучше удалить или заменить.

4. **Кто пишет и что предлагает?**
   - Type: Long text
   - Required: yes
   - Help text: Коротко: работодатель, клиент, партнер, знакомый, неизвестный аккаунт и т.д.

5. **Что вас тревожит?**
   - Type: Multiple select
   - Required: yes
   - Options:
     - Деньги / предоплата / комиссия
     - Документы / паспорт / карта
     - Ссылки / файлы / доступы
     - Слишком срочно
     - Слишком шаблонно
     - Похоже на подмену человека
     - Манипуляции / давление
     - Не могу сформулировать

6. **Нужен безопасный вариант ответа?**
   - Type: Multiple choice
   - Required: yes
   - Options:
     - Да
     - Нет

7. **Согласие с ограничениями**
   - Type: Checkbox
   - Required: yes
   - Text: Я понимаю, что это не юридическая консультация, не психологический диагноз и не гарантия мошенничества. Это анализ текстовых признаков риска.

## Confirmation Message

Спасибо. Если в заявке достаточно контекста, ответ придет на указанный контакт. Первые 3 короткие проверки для одного контакта бесплатны.
